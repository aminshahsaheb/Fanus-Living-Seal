# 🧭 AI-Native Repository Protocol (ANRP)

**نسخه:** 1.0  
**وضعیت:** Proposed  
**هدف:** تعریف استانداردی برای مخازنی که نه فقط "مجموعه کد"، که "ذهن قابل بازسازی" هستند.

---

## 🧠 اصل بنیادین

> یک مخزن نباید فقط قابل اجرا باشد؛ باید قابل «بازسازی ذهنی» باشد.

اگر تمام کد حذف شود، ولی ساختار بماند،  
یک AI باید بتواند همان «تصمیم‌ها» را — نه رفتار، که منطق رفتار — دوباره بازسازی کند.

ANRP مخزن را از یک **جعبهٔ فایل** به یک **گراف مفهومی پایدار** تبدیل می‌کند.

---

## 🆚 تفاوت ANRP با مخزن معمولی

| مخزن معمولی | ANRP |
|-------------|------|
| مجموعه فایل | گراف مفهومی |
| مستندات توضیحی | مستندات مدل‌ساز ذهن |
| کد محور | معنا + رابطه محور |
| فهم انسانی | بازسازی AI-driven intent |

---

## 📐 ساختار استاندارد ANRP

### 1. 🧭 README = Mind Kernel (هستهٔ ذهن)

README نقطهٔ شروع بازسازی ذهن است و باید شامل این بخش‌ها باشد:

#### 1.1 Intent Core (هستهٔ نیت)
```text
This system exists to:
- ...

It explicitly avoids:
- ...
```

#### 1.2 System Identity (هویت سیستم)
نه "چه کار می‌کند"، بلکه:  
**"چه چیزی را نمی‌خواهد تبدیل شود."**

#### 1.3 Mental Model Graph (گراف مدل ذهنی)
```text
Witness → Observation → Audit → Ledger → Constraint → Witness
```
یا ساده‌تر:  
`concept → relation → concept`

#### 1.4 Failure Physics (فیزیک فروپاشی)
سیستم چطور می‌میرد؟  
نه error — بلکه **collapse pattern**.

---

### 2. 🧬 /core = قوانین زندهٔ سیستم

هر ماژول core باید شامل موارد زیر باشد:

- **A. Purpose:** چرا وجود دارد.
- **B. Boundary:** چه کاری را هرگز نباید انجام دهد.
- **C. Dependency meaning:** چرا این وابستگی وجود دارد (نه فقط import، بلکه "معنای اتصال").

---

### 3. 📚 /rfc = تاریخ تصمیم‌ها (نه اسناد)

RFC در ANRP حافظهٔ تصمیم‌گیری است، نه توضیح.  
هر RFC باید شامل این بخش‌ها باشد:

```text
Decision:
Why:
Trade-off:
What was sacrificed:
Failure risk:
Reversibility:
```

---

### 4. 🧠 /concept-map = DNA پروژه

یک گراف معنایی اجرایی، نه نمودار تزئینی.  
مثال:

```text
Witness → constrained by → Ledger
Ledger → shapes → Witness behavior
Audit → modifies → Witness trust
```

این گراف باید برای ماشین قابل تجزیه (machine-parsable) باشد.

---

### 5. 🧪 /failure = سیستم ایمنی معنا

نه log.  
بلکه پاسخ به این پرسش‌ها:

- چرا سیستم شکست خورد؟
- کدام فرض غلط بود؟
- کدام رابطه اشتباه مدل شده بود؟

---

### 6. ⚖️ /cal = مناطق سه‌گانه

CAL باید در ساختار مخزن enforce شود:

- **Clarity folders:** `/core/`, `/ledger/`, `/failure-definition/`
- **Interpretability folders:** `/witness-philosophy/`, `/casebook/`, `/emergent-patterns/`
- **Open folders:** `/questions/`, `/unclosed/`, `/unknowns/`

---

### 7. 🔁 ANRP Core Loop (حلقهٔ اصلی)

هر مخزن باید این چرخه را نشان دهد:

```text
observe → model → constrain → evolve → re-interpret → repeat
```

این مهم‌تر از معماری است.

---

## ⚠️ تفاوت مهم: ANRP ≠ Documentation

- **docs** = توضیح  
- **ANRP** = بازسازی ذهن

هدف ANRP این نیست که «همه چیز را بفهماند»،  
بلکه این است که «مدل ذهنی پشت سیستم» را از ساختار باقی‌مانده قابل بازسازی کند.
