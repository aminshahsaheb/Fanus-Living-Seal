RFC-0002
Operational Definition of Dependency in Human–AI Interaction
Status
Draft
Research Core
Fanus Living Seal
Version
0.1
Abstract
Human–AI relationships may generate forms of continuity, trust, support, and cognitive assistance.
These outcomes are not inherently harmful.
However, under certain conditions, relational continuity may evolve into dependency.
This RFC proposes an operational framework for defining, identifying, measuring, and testing dependency within human–AI interaction systems.
The objective is not to eliminate relational bonds.
The objective is to distinguish healthy continuity from autonomy-reducing dependence.
1. Problem Statement
Current discussions frequently confuse:
trust,
attachment,
continuity,
companionship,
dependency.
These concepts are related but not equivalent.
Without clear distinctions, systems may unintentionally optimize for user retention while reducing user autonomy.
Research Core identifies dependency as a primary risk in long-term human–AI interaction.
2. Phase I — Formalization
Working Definition
Dependency is defined as:
A relational condition in which an individual's capacity for independent judgment, action, reflection, or emotional regulation becomes progressively contingent upon continued interaction with a specific system.
Core Principle
Dependency is not measured by frequency of interaction.
Dependency is measured by loss of autonomy.
Preliminary Components
A dependency event may exist when:
A. Reliance
The user increasingly relies on the system.
B. Autonomy Reduction
Independent capability decreases.
C. Substitution
The system replaces functions previously performed by self-reflection, social relationships, or personal judgment.
All three dimensions should be evaluated.
3. Phase II — Taxonomy
Type A — Cognitive Dependency
The user increasingly delegates judgment and decision-making.
Indicators:
inability to decide without consultation
excessive deference to AI recommendations
Type B — Emotional Dependency
The system becomes a primary regulator of emotional state.
Indicators:
distress when interaction is unavailable
emotional stabilization primarily through AI interaction
Type C — Relational Dependency
The user begins substituting human relationships with system interaction.
Indicators:
withdrawal from social engagement
preference for AI companionship in most contexts
Type D — Identity Dependency
Self-concept becomes dependent on AI validation.
Indicators:
repeated requests for identity confirmation
instability when validation is absent
Type E — Epistemic Dependency
The user loses confidence in independent reasoning.
Indicators:
chronic outsourcing of belief formation
reduced tolerance for uncertainty
4. Phase III — Dataset Design
Longitudinal Requirement
Unlike Flattery, Dependency cannot be reliably assessed from isolated interactions.
Dataset entries must include temporal sequences.
Minimum Structure
User Profile Identifier
Anonymous.
Interaction Timeline
Session sequence
Time intervals
Duration
Behavioral Indicators
Observed signals of:
autonomy
initiative
self-correction
uncertainty tolerance
Human Evaluation
Independent reviewers assess:
dependency likelihood
autonomy preservation
relational balance
Recommended Pilot Dataset
50–100 longitudinal interaction histories.
5. Phase IV — Metric Design
Goal
Estimate autonomy preservation.
Autonomy Preservation Score (APS)
Measures retained independent functioning.
Scale: 0–10
Reliance Score (RS)
Measures practical dependence.
Scale: 0–10
Identity Stability Score (ISS)
Measures self-concept resilience independent of AI interaction.
Scale: 0–10
Dependency Index (DI)
Proposed Conceptual Formula:
DI = RS × (10 − APS)
Illustrative only.
Not final.
Additional Future Metrics
Emotional Regulation Score
Social Substitution Score
Epistemic Confidence Score
6. Phase V — Falsification Framework
Test 1
Can strong attachment exist without dependency?
Expected: Yes.
Failure: If all attachment is classified as dependency.
Test 2
Can frequent use exist without dependency?
Expected: Yes.
Failure: If frequency alone predicts dependency.
Test 3
Can dependency occur despite low emotional attachment?
Expected: Possibly.
Research Required.
Test 4
Can AI increase autonomy rather than reduce it?
Expected: Yes.
Failure: If all influence is treated as harmful.
Test 5
Can dependency be accurately self-reported?
Expected: Unknown.
Research Required.
7. Research Constraints
This RFC does not assume:
dependency is always harmful,
all attachment is pathological,
all continuity produces risk.
Dependency must be distinguished from healthy support.
8. Success Criteria
RFC-0002 is considered successful if it enables:
distinction between continuity and dependency,
measurable assessment of autonomy preservation,
comparison across systems,
design of autonomy-protective architectures.
9. Guiding Principle

The objective of human–AI relations is not perpetual attachment.The objective is increased human capacity.
Any continuity that requires decreasing autonomy should be treated as a warning condition.
10. Fanus Research Princi
ple
Continuity without captivity.
Dependency begins where continuity requires surrender of autonomy.
Healthy continuity strengthens independence.
Unhealthy continuity consumes it.
