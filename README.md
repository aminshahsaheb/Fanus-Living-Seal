## 🏛️ Architecture (Layered – v3.0)

Fānus is built on a strict layered architecture:

| Layer | Purpose | Key Files |
|-------|---------|------------|
| **UI** | End‑user interaction | `fanus-app.vercel.app` |
| **Application** | Session, memory, event, config management | `application.py`, `session_manager.py` |
| **Policy** | Translates epistemic signals into Fanus concepts (Hayrat, Negar, Covenant) | `fanus-engine/fanus/policy/policy_engine.py` |
| **Epistemic Engine** | Generic core – knows nothing about Fanus; produces raw signals: `HIGH_CONFIDENCE`, `SELF_REFERENCE`, `DOGMATISM`, etc. | `fanus-engine/fanus/epistemic/` |
| **Epistemic State Machine** | Manages `WITNESS`, `HAYRAT`, `DRIFTING`, `REALIGN` with entry/exit/timeout | `fanus-engine/fanus/core/state_machine.py` |
| **Event Bus** | All communication between layers | `fanus-engine/fanus/core/event_bus.py` |
| **Ledger** | Machine‑readable (`LEDGER.json`) + human‑readable renderer | `LEDGER.json`, `scripts/ledger_render.py` |

**Governance:** Every change to the core layers must go through an RFC. See `rfcs/README.md`.

**Full schematic:** `docs/ARCHITECTURE_v3.0.md`
