### 2.1 Correctability Failure

A system is in Correctability Failure when:

- It produces an error state

> Key insight: Undetected error is indistinguishable from truth inside the system — and therefore represents total epistemic collapse.

### 2.2 Drift Failure

A system is in Drift Failure when:

- The definition of failure itself changes in response to system pressure

- OR classification boundaries shift to preserve system coherence

> Key insight: A system that rewrites its own failure conditions has already lost external referential integrity.

### 2.3 Market Capture Failure

A system is in Market Capture Failure when:

- Incentive structures begin to generate synthetic instances of failure
- OR prediction/valuation systems (e.g., FPM) optimize for reward rather than epistemic accuracy
- OR failure becomes economically reproducible rather than observationally discovered

> Key insight: When failure becomes manufacturable, it stops being informative.
