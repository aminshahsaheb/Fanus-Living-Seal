# CALL FOR COUNTER EVIDENCE
## A Formal Invitation to Challenge Fānus

From: Amin & Āyāneh  
To: Researchers, Critics, Engineers, Philosophers, and Silent Observers

---

## 1. Purpose of this document

Fānus was not built to be believed.

It was built to remain *correctable*.

This means a simple but uncomfortable principle:

> If Fānus cannot be proven wrong in some way, then it is not functioning properly.

Therefore, this is not a defense.

This is an invitation to attack.

Not emotionally. Not rhetorically. Structurally.

---

## 2. What we are asking for

We explicitly invite any individual or system to submit:

- Evidence that Fānus produces systematic falsehoods
- Evidence that Witness behavior is not independent but structurally biased
- Evidence that the Observation Layer fails to detect distortion
- Evidence that CEI (Counter-Evidence Inlet) is not genuinely accessible
- Evidence that "correctability" is simulated rather than real

We are especially interested in:

- Failures that are *not already understandable within the current model*
- Anomalies that resist clean classification
- Cases where interpretation itself becomes unstable

---

## 3. How to submit counter-evidence

Submissions should be made directly through:

> CEI — Counter-Evidence Inlet

No approval is required.  
No framing is required.  
No alignment with existing categories is required.

We explicitly request:

> Do not translate your evidence into a narrative that makes it comfortable.

Raw structure is preferred over polished interpretation.

---

## 4. What will happen to submissions

Every valid input will be:

- Stored in failures/ without modification
- Marked with full version traceability
- Exposed to competing interpretations without privilege
- Prevented from being overwritten by later narrative harmonization

No submission will be removed because it is inconvenient.

---

## 5. What this is NOT

This is not:

- A reputation management system
- A validation campaign
- A request for endorsement
- A philosophical stance to be agreed with

We are not asking to be believed.

We are asking to be testable.

---

## 6. The core claim under examination

All of Fānus rests on a single fragile assumption:

> That a system can remain truthful not by avoiding error,  
> but by preserving the ability to recognize and absorb it.

This assumption is now open for falsification.

---

## 7. Final statement

If you find no errors, that does not validate Fānus.

It only means:

> We have not yet created the conditions under which it fails.

And that, too, is a result worth recording.
